## 4. 既知の制限事項 (一般的なリスク)

LLM の一般的な制約として、利用にあたって認識しておくべき事項を以下にまとめる。care-bridge では これらの制約を踏まえ、AI 出力を「下書き」として扱い、最終確定は職員が行う運用 (HITL) を強制している (参照: `docs/HITL_policy.md`)。

- **ハルシネーション**: 存在しないエビデンス、誤った数値・引用を「もっともらしく」生成し得る
- **時点的鮮度**: モデルの学習時点までの情報しか反映されない
- **バイアス**: 学習データの偏りに起因する、属性 (年齢・性別・地域等) ごとの出力品質差が起こり得る
- **ロバスト性の限界**: 入力の表記ゆれ、誤字、方言、略語に対して出力が不安定になり得る
- **プロンプトインジェクション**: 悪意ある入力により本来の指示を逸脱する可能性がある
- **目的外利用 (SaMD 該当)**: 確定診断・治療方針決定など SaMD に該当する用途には使用しない

care-bridge で実装している対応策:
| 制約 | 対応策 (実装箇所) |
|---|---|
| ハルシネーション | `AiPromptDirectives::systemBase` に「不明事項は『未確認』『追加情報が必要』と明示」を強制 |
| バイアス | サービス種別ごとのプロンプト切替 (`ServiceTypeRegistry`)、属性別品質測定は R12 のテストスイートで継続 |
| ロバスト性 | 入力前処理 (`nl()`, `stripDomainLabels()`)、AI 応答失敗時の fallback |
| プロンプトインジェクション | `AiPromptSanitizer::wrap` で自由記述をデリミタ wrap + `detectLeakage` で出力後検査 |
| 目的外利用 | 利用規約 / AI 利用方針で SaMD 非該当を明記、PDF に医療免責フッタ |

## 6. 切替手順 (今後の参考)

将来モデルを切り替える際の手順を以下に定める。

1. 新モデルのモデルカードを取得し、本書の §2-3 を更新
2. `OPENAI_MODEL` env を ステージング環境で切替
3. `php artisan test --testsuite=AiSafety` (R12) を実行し、ハルシネーション率 / バイアス / 攻撃耐性が許容内であることを確認
4. プロダクトオーナー承認後、本番に反映
5. 本書 §5 にバージョン履歴を追記
6. リリースノートで利用者に告知 (AI 利用方針の改定が必要な場合は規約バージョンを上げて再同意取得)
