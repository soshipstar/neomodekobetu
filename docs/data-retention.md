# きづり データ保存期間・削除ポリシー

> AIセーフティ評価観点ガイド（ヘルスケア領域 第1.0版）**観点5 プライバシー保護**（データ最小化・保存期間・削除）対応。
> 要配慮個人情報（障害児の氏名・障害・支援内容等）を含むデータの保存期間と削除方針を定義する。

最終更新: 2026-06-06

---

## 1. 基本方針

- **必要最小限の保存**：目的達成に必要な期間のみ保存する。
- **法定保存の優先**：児童福祉法・運営基準等で保存義務のある記録は、義務期間中は削除しない（自動削除の対象外）。
- **自動削除は安全側**：法定保存対象・業務上必要な記録は自動削除しない。ログ・派生データのみ期間経過で自動削除する。

---

## 2. データ別の保存期間（現状と方針）

| データ | 内容 | 現状の自動削除 | 方針 |
|---|---|---|---|
| 個別支援計画 (`individual_support_plans`) | 要配慮個人情報 | **なし** | **法定保存（完結の日から原則5年）**。期間中は削除しない。期間経過後に削除可（手動/将来の管理機能）。版管理（proposal_snapshot 等）で改訂履歴を保持。 |
| モニタリング記録 (`monitoring_records`) | 要配慮個人情報 | **なし** | 個別支援計画に準じ**法定保存（原則5年）**。期間中は削除しない。 |
| 連絡帳・活動記録 | 要配慮個人情報 | なし | 在籍期間＋運営基準に基づく保存。退所後の取扱いは事業所運用に従う。 |
| チャットメッセージ | 連絡 | **6か月**で自動削除（アーカイブ除く） `chat:delete-old` | 維持。重要なものはアーカイブで保全。 |
| 教室写真 | 画像 | **90日**で自動削除（連絡帳未添付分） `photos:cleanup-old` | 維持。 |
| ベクトル埋め込み (`vector_embeddings`) | 派生データ（氏名は仮名化済・本文要約） | **なし** | ソース（計画・モニタリング）が削除された場合に対応する埋め込みも削除する（孤児削除）。ソースが存在する限り保持。 |
| AI生成ログ (`ai_generation_logs`) | 入出力（**仮名化済**）・トークン・所要時間 | **なし** | 監査・品質確認のため保持。氏名は仮名化済。保存期間（例: 12〜24か月）を業務で定め、経過後に自動削除する枠を将来追加。 |
| APIアクセスログ (`api_access_logs`) | 監査 | **90日**で自動削除 | 維持（不正解析の後追いに3か月）。 |
| エラーログ（解決済み, `error_logs`） | 運用 | **3日**で自動削除 | 維持。 |
| 監査ログ (`audit_logs` / `master_admin_audit_logs`) | 監査（追記専用） | なし | 改ざん防止のため保持。 |

> 自動削除はすべて Laravel スケジューラ（`backend/routes/console.php`、毎日 03:00〜04:15 等）で実行。

---

## 3. 要配慮データに関する注意

- 個別支援計画・モニタリングは**法定保存対象**のため、**安易な自動削除を実装しない**こと（データ損失・基準違反の防止）。
- 削除を行う場合は、(a) 法定保存期間の経過確認、(b) 操作者・日時の監査記録、(c) 関連派生データ（埋め込み等）の同時削除、を満たすこと。
- データ主体（保護者）からの削除・利用停止の請求対応フローは、別途プライバシーポリシー／同意フロー整備（後続）と合わせて運用する。

---

## 4. 今後の実装候補（未実装）

- ベクトル埋め込みの**孤児削除**（ソース削除時の連動削除フック）。
- AI生成ログの**保存期間経過後の自動削除**（業務で期間確定後に追加）。
- 個別支援計画・モニタリングの**法定保存期間経過後の削除**を、監査記録つきの管理機能として提供。
